基于设计文档中的架构规划，该系统采用了**前后端分离**的现代敏捷开发架构。后端专注于高性能图像计算与任务调度，前端专注于流畅的交互体验与可视化展示。

以下是具体的技术选型介绍：

### 1. 后端技术栈 (Backend)
后端是系统的“大脑”，负责处理繁重的图像算法和数据管理。

*   **核心框架：Python 3.11 + FastAPI**
    *   **Python**：图像处理和科学计算的首选语言，拥有最丰富的生态。
    *   **FastAPI**：高性能的异步 Web 框架。利用其异步特性（AsyncIO），支持高并发请求，特别适合通过 WebSocket 推送实时的“手动调参预览”流，且自动生成 API 文档方便前后端联调。
*   **图像算法库：OpenCV + NumPy + SciPy**
    *   这是实现颜色校正（如 Gray World）、对比度增强（CLAHE）和去噪算法的核心计算引擎。
    *   *扩展性*：预留了 **PyTorch / ONNX Runtime** 接口，用于挂载未来的深度学习模型（如 WaterNet）。
*   **异步任务队列：Celery + Redis**
    *   **痛点解决**：水下图像（特别是 4K/8K 图）处理耗时较长。
    *   **作用**：将大分辨率图像处理、批量任务和复杂的指标计算（UIQM/UCIQE）从主线程剥离，放入后台队列异步执行，防止阻塞 Web 服务，确保界面不卡顿。
*   **数据存储：PostgreSQL + MinIO**
    *   **PostgreSQL**：存储任务元数据、评估指标报告和用户配置。
    *   **MinIO**：高性能对象存储（兼容 S3），专门用于存储原始图片、修复后的图片和生成的 PDF 报告文件。

---

### 2. 前端技术栈 (Frontend)
前端是系统的“门面”，重点在于实现类似桌面软件的丝滑操作感，特别是图像对比和参数调整部分。

*   **核心框架：React 18 + TypeScript + Vite**
    *   **React**：组件化构建页面，生态丰富。
    *   **TypeScript**：强类型语言，确保处理复杂的图像数据结构（如直方图数据、ROI 坐标）时不出错。
    *   **Vite**：极速构建工具，提升开发体验。
*   **状态管理：Zustand + TanStack Query**
    *   **Zustand**：轻量级状态管理，专门用于处理**手动调参面板**的高频状态更新（如拖动滑块时），配合防抖（Debounce）机制与后端通信。
    *   **TanStack Query**：管理服务端数据（如上传进度、任务列表、报告数据），处理缓存和自动刷新。
*   **UI 组件与样式：Tailwind CSS + Radix UI**
    *   **Tailwind CSS**：原子化 CSS，快速复刻设计文档中的渐变、卡片浮动和阴影效果。
    *   **Radix UI**：无头组件库，提供无障碍支持的 Slider（滑块）、Dialog（弹窗）等底层交互能力，保证组件的高质量。
*   **图形与可视化：Konva / Fabric.js + ECharts**
    *   **Konva/Canvas API**：核心交互组件。用于实现**双图对比**（擦除式滑块）、**局部放大镜**、以及在图片上绘制标注。比操作 DOM 性能更高。
    *   **Apache ECharts**：用于绘制评估报告中的专业图表（如 RGB 通道直方图、雷达图），展示修复前后的指标提升。

### 3. 总结
这套技术组合是一个**计算密集型**应用的典型架构：
*   **前端**负责“轻快”的交互和 Canvas 渲染；
*   **后端**负责“厚重”的矩阵运算；
*   **中间**通过 WebSocket 和消息队列（Redis）解耦，保证了系统既能处理复杂的图像算法，又能给用户提供实时、流畅的操作反馈。
